NuxtJs+Vuetify 静态博客搭建（1）

创建NuxtJs项目

准备环境

  • 拉取Docker镜像
terminal
docker pull node:20.3.0-bullsey

如果速度较慢，可设置先设置镜像源，然后拉取。

  • 设置镜像源 
{
  "registry-mirrors": [
    "https://registry.docker-cn.com",
    "https://cr.console.aliyun.com"
  ]
}
  • 创建容器
terminal
docker run -dt --privileged --gpus all ^
-v <path-to-nodejs-projects>:/root/nodejs-projects/ ^
--name nodejs-dev ^
node:20.3.0-bullseye

新建和运行项目

  • 容器中，新建nuxt项目，安装依赖的包
container terminal
cd /root/nodejs-projects/
npx nuxi init <project-name>
cd <project-name>
npm install
  • 运行项目，用于开发调试。
container terminal
npm run dev -- -o

部署项目

  • 设置baseURL

部署在github page后，博客路径为<域名>/<仓库名>/，需要设置nuxtjs，是的路由中包含仓库名。

nuxt.config.ts
export default defineNuxtConfig({  
  app: {
    baseURL: '/blog/'
  }
})
  • 生成静态网页，用于部署。 执行下面的语句，输出在.output/public目录。
container terminal
npx nuxi generate

  • 在.output/public目录添加空文件.nojekyll 否则，部署后无法显示样式。 将.output/public作为一个本地仓库push到github仓库或其他静态网站托管平台。
  • github仓库开启page服务。
